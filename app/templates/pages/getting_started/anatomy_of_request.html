Anatomy of a request

All of our API requests are submitted as a URL. For example: 

https://acceptor.paypoint.net/rest/transactions/{instId}/{resourceId}/action

The request is broken down as:

Base URL /Resource Type /InstId /Resource ID /Action
The base URL for our rest services. The type of resource you are submitting the request for. The ID for your installation as provided by our Client Management team. The ID for a resource either being fetched or operated on. This isn't present when creating a new resource. The action being performed.
Examples: 
https://acceptor.paypoint.net/rest
https://hosted.paypoint.net/rest
Examples: 
/transactions
/customers
/sessions
Examples: 
/<CustomerID>
/<TransactionID>
/<CustomerID>/cards/<tokenID>
Examples: 
/payment
/refund

The body of the requests and responses uses the JSON format. All our requests are either POST or GET. When the operation is successful the response code will be a 200, or 201 in the case where a new resource is created. Your credentials need to be sent with every request in the header using the HTTP Basic format, for example: 

Accept: application/json
Content-Type: application/json
