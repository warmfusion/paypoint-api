Outlet locator

You may wish to offer customers a lookup service to locate their nearest PayPoint outlet – remember, there are over 25,000 in the UK, with over 95% of the population within one mile (urban areas) and five miles (rural areas) of a participating retailer.

PayPoint offer a centralised web service for locating nearest PayPoint outlets. The web service definition (WSDL) is located here: http://locator.paypoint.com:61001/sitelocator.asmx?wsdl

For locating outlets which can take payment for PayCash transactions, you need to use the GetNearestSitesType2 method. It will search in all branded PayPoint terminal locations, since all participate in this service. Input parameters are as follows:

  • searchCriteria (the value you want to search by, for example a postcode, town or city, locale, or full address)
  • searchType (a numeric value denoting the type of search to perform, 0 = PayPoint automatically determines based on the searchCriteria supplied, 2 = postcode, 3 = postcode district, 4 = town/city)
  • maxRecords (the number of nearest outlets to be returned, from between 1 and 20)
  • openNow (an optional flag to indicate we should only return outlets open at the time of search – a real ‘live’ look-up)

For each outlet identified by proximity, the returned data structure includes detailed information on the site including distance from the search location given in searchCriteria, the full site address and the full weeks opening hours of each location.

PayPoint operate many different services via its retail outlets – including bill payment and mobile top-up. You should disregard any other optional parameters mentioned in the WSDL as these are simply different searches for other PayPoint services.
