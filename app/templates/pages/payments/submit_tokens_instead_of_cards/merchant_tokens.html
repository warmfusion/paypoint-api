Merchant tokens

If you’re using the API product, upon successful completion of your customer’s first payment, we’ll return a token and a customer reference in the response. Either of these can be used for future Payment, Authorisation or Payout requests in place of the customer’s card number. (PAN – Primary Account Number). All you need to do is either provide the customer reference or the token itself for each subsequent request.

Where a customer has registered multiple cards, you can also request a Payment, Authorisation or Payout from the default card.
